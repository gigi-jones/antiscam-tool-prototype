# Anti-Scam Policy Tool — Stage 1 Assessment Prototype

An interactive walkthrough of the Stage 1 Assessment module of a proposed digital policy tool for LMIC policymakers working on authorised payment fraud.

## Purpose

This prototype is a **funder-facing concept demonstration**, not a working tool. It is designed to communicate:

- What each module of the Stage 1 Assessment asks of users
- **The output produced at each stage** — this is the core of the demonstration
- How the assessment is structured to serve any entry-point (financial authority, senior political figure, cross-government coordinator, or development partner)
- How the final output — the "living map" — provides a common basis for discussion, action, and scaling

The prototype shows representative sample questions per module rather than the full questionnaire. In production, each module contains detailed branching assessments that adjust based on prior answers.

## Context

The tool is being developed to accompany RUSI CFS's research on authorised payment fraud policy, with potential funding support from the Gates Foundation. It is grounded in:

- A comprehensive literature review and 34 stakeholder interviews across 16 countries
- Validation input from expert reviewers including Nick Bourke (Consumer Bankers Association / Roadmap to a National Strategy)
- A design philosophy that makes the human work of building effective fraud responses easier — not a substitute for it

## How to use

Open `index.html` in any modern browser. Navigate through the modules using the left sidebar. Each module shows:

1. A representative sample question
2. A helper note on the diagnostic logic
3. **The output that module produces** — displayed prominently in the accent-coloured section

## Structure

- **Overview** — design principles and module summary
- **Five Policy Lenses** (Modules 1–5) — fraud landscape, stakeholders, institutions, legal, political economy
- **Cross-Cutting Layers** (Modules 6–7) — fraud chain overlay, information flows
- **Enabler Diagnostics** (Modules 8–10) — operationalisation, coordination, unintended consequences
- **Final Output** (Module 11) — the living map
